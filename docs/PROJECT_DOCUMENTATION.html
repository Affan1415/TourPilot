🚀 Tour Booking System

A complete FareHarbor clone built with Next.js and Supabase - Save 8% on every booking by owning your platform.

Next.js 14 Supabase Stripe Twilio TypeScript

1. Introduction

Project Overview

This project is a comprehensive tour and activity booking platform designed to replace FareHarbor and similar booking systems. By building your own platform, you eliminate the 6-8% commission fees charged by third-party booking systems.

Key Objectives

  • Zero Platform Fees: Only pay payment processor fees (~2.9%)
  • Full Control: Own your customer data and booking system
  • Custom Features: Tailor the system to your specific needs
  • Scalability: Built on modern, scalable infrastructure

Target Users

Tour Operators

Boat tours, walking tours, adventure activities, and experiences

Captains & Guides

On-the-ground staff managing check-ins and customer experience

Customers

Tourists and locals booking tours and activities online

6. API Endpoints

Update tour details (admin)

GET /api/availabilities?tour_id=&date=
Get available time slots for a tour on a date

Create availability slots (admin)

Create a new booking

Get booking details by ID or reference
PUT /api/bookings/[id]/check-in

PUT /api/bookings/[id]/cancel

10. Captain Manifest (Attendance Sheet)

Overview

The manifest is a daily attendance sheet that captains and guides use to manage check-ins, track guests, and ensure everyone has signed their waivers before boarding.

Manifest Features

📅 Daily View

See all bookings for today or any selected date

🚢 Per-Tour Breakdown

Filter by tour type and time slot

👥 Guest Count

Total souls on board including walk-ups

✅ Check-in Status

One-tap check-in for each guest

📝 Waiver Status

Color-coded waiver indicators (green/orange/red)

🖨️ Print View

Print-friendly format for offline use

Manifest Data Displayed

Column Description
Booking Ref Unique booking reference number
Guest Name Primary guest name
Party Size Number of guests in booking
Contact Phone number for quick contact
Waiver Status / /
Check-in Checkbox to mark as checked in
Notes Special requests or notes

Captain Workflow

  1. Open manifest on phone/tablet
  2. Select today's date and tour
  3. Review guest list and total count
  4. As guests arrive:
    • Find their name on the list
    • Check waiver status (must be green)
    • If red/orange → guest signs waiver on device
    • Tap to check them in
  5. Handle walk-ups (add to manifest)
  6. Review final count before departure

13. Staff Workflow

Captain/Guide Daily Routine

  1. Start of Day: Log in to manifest app
  2. Review Schedule: Check today's tours and guest counts
  3. Pre-Tour Prep: Review special requests and notes
  4. Guest Arrival:
    • Greet guests by name
    • Verify waiver status on manifest
    • Handle on-site waiver signing if needed
    • Check in each guest
  5. Handle Walk-ups: Add to manifest, process payment, sign waiver
  6. Final Count: Verify all guests checked in before departure
  7. Mark No-shows: Update status for absent guests

Staff Roles & Permissions

Role Permissions
Admin Full access: tours, bookings, pricing, staff, reports, settings
Manager Manage bookings, view reports, assign staff, limited settings
Captain/Guide View assigned manifests, check-in guests, handle waivers
Front Desk Create bookings, process payments, view manifests
